    ```typescript
    const [status, setStatus] = useState<'idle' | 'running' | 'done' | 'error'>('idle');

    const [errorMessage, setErrorMessage] = useState('');
    ```

    **runReflection function:**
    1. Set status to 'running', clear rawOutput and errorMessage
    2. `const response = await fetch('/api/reflection/run', { method: 'POST' });`
    3. If !response.ok:
       - If 429: setErrorMessage('Reflection already run in the last 24 hours')
       - Else: setErrorMessage('Failed to run reflection')
       - setStatus('error'); return
    4. Read stream:
       ```typescript
       const reader = response.body!.getReader();
       const decoder = new TextDecoder();
       while (true) {
         const { done, value } = await reader.read();
         if (done) break;
         setRawOutput(prev => prev + decoder.decode(value, { stream: true }));
       }
       setStatus('done');
       ```

    **parsedReport memo:**
    When status === 'done', parse rawOutput as ReflectionReport. Use try/catch — if parse fails, show raw text as fallback.
    ```typescript
    const parsedReport = useMemo(() => {
      if (status !== 'done' || !rawOutput) return null;
      try {
        const cleaned = rawOutput.replace(/^```(?:json)?\n?/m, '').replace(/\n?```$/m, '').trim();
        return JSON.parse(cleaned) as ReflectionReport;
      } catch {
        return null;
      }
    }, [status, rawOutput]);
    ```

    **Render:**
    1. Header with "System Reflection" title
    2. If `reflectionDue` and status === 'idle': show amber badge "New activity since last reflection"
    3. Button: "Run Reflection" (disabled when status === 'running')
       - During running: show spinner or "Analyzing..." text
    4. If status === 'error': show error message in red
    5. If status === 'running': show rawOutput in a `<pre>` or monospace div, streaming live
    6. If status === 'done' and parsedReport:
       Render 5 expandable sections using `<details><summary>` elements:
       - **Patterns** (`parsedReport.patterns`): List of bullet points
       - **Contradictions** (`parsedReport.contradictions`): Each with description, optionally showing node_ids
       - **Coverage Gaps** (`parsedReport.coverage_gaps`): List of bullet points
       - **Trajectory** (`parsedReport.trajectory`): Single paragraph
       - **Recommendations** (`parsedReport.recommendations`): Each recommendation as a row with:
         - Text description
         - Action button (if action_type is not null):
           - `create_node`: Link to `/capture/new` with text "Create Node"
           - `review_node`: Link to `/capture/${target_node_id}/review` with text "Review"
           - `flag_tension`: No link (just text "Flag Tension" — no dedicated UI yet)
           - `link_node`: No link (just text "Link Node" — deferred per RESEARCH.md)
    7. If status === 'done' and !parsedReport: show raw text as fallback

    **Styling:** Match existing weekly review page patterns:
