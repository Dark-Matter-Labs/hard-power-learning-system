## Spacing Scale

Tailwind classes map to these values. GoalSpacePanel uses the same scale as NodeDetailPanel:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | `gap-1`, `mt-1`, `gap-1.5` — count label gaps |
| sm | 8px | `p-2` — outcome card inner padding; `mb-2` — section header bottom |
| md | 16px | `p-4` — panel outer padding |
| lg | 24px | not used |

Exceptions: `top-[49px]` (49px top offset matching app chrome height — not a spacing scale value, fixed to match existing panel positioning).

Spacing exceptions (non-scale values used in Component Anatomy):

| Class | Value | Justification |
|-------|-------|---------------|
| `mt-0.5` | 2px | Sub-grid micro-alignment for unicode symbol baseline — 2px corrects visual misalignment without affecting layout |
| `mt-1.5` / `gap-1.5` | 6px | Intermediate breathing room between card title and count row — midpoint between xs (4px) and sm (8px) to avoid crowding |

## Component Anatomy

### GoalSpacePanel container

```
absolute right-0 top-[49px] bottom-0 w-72 bg-gray-950 border-l border-gray-800 p-4 overflow-y-auto
```

Width: 288px (w-72). Positioned identically to NodeDetailPanel — no z-index change needed; CommitmentPanel is on the left (left-0) so there is no overlap.

### Header row

```
flex items-center justify-between mb-3
```

- Left: `<NodeTypeBadge nodeType={node.node_type} />` — renders "goal space" badge with `bg-[#0F6E56] text-white text-xs px-2 py-0.5 rounded-full` (existing component, no changes)
- Right: `<button onClick={onClose} className="text-gray-600 hover:text-gray-400 text-lg" aria-label="Close goal space panel">×</button>` — `text-lg` (18px) applies only to this single `×` character; it is not part of the content type scale

### Node title

```html
<h3 className="text-sm font-bold text-gray-200 mb-4">{node.title}</h3>
```

`mb-4` (16px) separates the title from the trigger outcomes section — slightly more breathing room than NodeDetailPanel's `mb-2` because this panel has no extraction data between title and content.

### Section header

```html
<div className="text-[10px] text-gray-600 uppercase mb-2">
  Trigger Outcomes ({outcomes.length})
</div>
```

### Empty state

```html
<p className="text-[10px] text-gray-700 italic">No trigger outcomes linked</p>
```

Rendered only when `outcomes.length === 0`. No icon, no CTA — Phase 2 is read-only.

### Outcome card

```html
<div className="mb-3 border border-gray-800 rounded p-2">
  <div className="flex items-start gap-2">
    <span className={`text-sm mt-0.5 ${colorClass}`}>{symbol}</span>
    <span className="text-[11px] text-gray-300 leading-tight">{outcome.title}</span>
  </div>
  <div className="flex gap-3 mt-1.5 pl-5">
    <span className="text-[10px] text-gray-600">{commitmentCount} commitment{commitmentCount !== 1 ? 's' : ''}</span>
    <span className="text-[10px] text-gray-600">{hunchCount} hunch{hunchCount !== 1 ? 'es' : ''}</span>
  </div>
</div>
```

`pl-5` (20px) on the count row aligns counts beneath the outcome title, clearing the symbol column.

## Status Symbol Contract

| Status | Symbol | Color class | Condition |
|--------|--------|-------------|-----------|
| `not_started` | ○ | `text-gray-600` | No connected commitments, hunches, or signals |
| `in_progress` | ◐ | `text-yellow-500` | At least one `assigned_to_outcome` commitment OR `targets_outcome` hunch OR `indicates_progress` signal exists |
| `met` | ◉ | `text-teal-400` | At least one `indicates_progress` signal with `status === 'promoted'` (Phase 2 simplification — no `content.outcome` check yet) |
| `blocked` | ✕ | `text-red-500` | Any connected source node has `status === 'falsified'` OR `status === 'suspended'` (checked first, overrides all) |

Status symbols are rendered inline as unicode characters at `text-sm` (14px) with `mt-0.5` to align with outcome title text. No hover or click behaviour in Phase 2 (badge click is Phase 5).

## Interaction Contract

| Trigger | Behaviour | Implementation note |
|---------|-----------|---------------------|
| User clicks a `goal_space` node in the graph | GoalSpacePanel renders, NodeDetailPanel is suppressed | Type-guard branch in `GraphOSSurface` render: `selectedNode.node_type === 'goal_space'` |
| User clicks × button | Panel closes | `onClose()` → `setSelectedNode(null)` in `GraphOSSurface` |
| User clicks a non-goal_space node while GoalSpacePanel is open | GoalSpacePanel closes, NodeDetailPanel opens | `selectedNode` state change naturally re-evaluates the branch |
| User clicks the graph canvas background | Panel closes (existing behaviour, no change needed) | `handleBackgroundClick` in `GraphOSSurface` already calls `setSelectedNode(null)` |
| Status symbol hover | No effect | Phase 5 adds badge click expansion — Phase 2 is static |
| Outcome card click | No effect | Phase 2 is read-only |
